
1. 微服务拓扑与重命名
   Lumina-UI (原前端)： 基于 React + Vite + TailwindCSS。负责处理复杂的视觉渲染（像素风 UI、PDF 联动、Zen Mode 状态管理）。
   Aether-Engine (原后端)： 基于 Python FastAPI。作为中枢引擎，负责调度 PDF 解析、向量/图数据库读写以及多智能体（Multi-Agent）对话路由。
2. 核心数据结构规范 (JSON Schema)
   后端与前端的通信，以及存入数据库的数据，必须严格遵守统一的 ZettelNote（原子笔记）JSON 格式,。这解决了以前数据混乱的问题。

3. Docker 单容器高可用部署 (Deployment)
   考虑到云端环境（如 ModelScope Studio）往往只开放一个对外端口，必须采用多阶段构建 (Multi-stage Build) 与进程管理器结合的方案 1。
   在 Dockerfile 中，使用 Node 镜像编译 Lumina-UI，然后将其打包进最终的 Python 镜像中。
   引入 supervisord 作为主进程（PID 1）。由它同时拉起 Nginx（监听 80 端口，代理静态文件）和 Uvicorn（运行 Aether-Engine，监听 8000 端口）,。
   在 Nginx 配置中，将 /api 路径的反向代理指向 http://localhost:8000。
