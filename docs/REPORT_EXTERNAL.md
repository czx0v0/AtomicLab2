# AtomicLab 对外汇报版（业务与产品价值）

## 1. 一句话定位

AtomicLab 是一个面向科研写作场景的 AI 工作台，把“读文献、做笔记、写论文”三段流程合并为一个可执行闭环。

## 2. 核心价值主张

- **效率提升**：从“检索-复制-粘贴”升级为“一键注入正文”
- **质量提升**：回答可引用、可回跳、可验证
- **体验提升**：从工具堆叠转向沉浸式写作舱（Notes/Atomic/Graph）

## 3. 关键能力（当前可演示）

### 3.1 文献理解与知识沉淀

- PDF 解析为结构化内容（章节、片段、知识卡片）
- 原子知识抽取（Axiom/Method/Boundary）
- 全局知识图谱与三元组视角
- **发现**：对接 ArXiv 与学术秘书收件箱；中文关键词可自动翻译为英文后再检索，提升可用性

### 3.2 智能检索与可解释问答

- 混合检索：向量 + BM25 + 图谱扩展
- 搜索结果下拉展示 + 命中标签（向量/图谱/关键词）
- 点击结果或聊天引用可直达页码并高亮原文

### 3.3 Agent 写作能力

- 支持“帮我写/润色/添加正文”等指令
- 助手不仅给答案，还可直接调用工具写入编辑器
- 人在回路：用户可继续改写与筛选
- 助手回复支持 **公式、表格、分条引用**；可与 **任务控制中心** 联动，一键进入「模拟评审」「返修」等写作场景
- **顶栏状态（非 Zen）**：展示连接/流式简要提示，与对话区合成气泡配合；首包到达前合成区有「正在生成」类占位，避免空白气泡与重复加载感

## 4. 典型使用流程（Demo 话术）

1. 加载 Demo 文献（全局单例解析，后续秒开）
2. 在搜索栏输入关键词，展示命中片段
3. 点击结果，自动跳转阅读页并高亮
4. 在写作舱切换到 Atomic/Graph，选择知识点
5. 点击 `[+ 插入正文]`，直接生成到左侧草稿
6. 让 Agent 继续润色，完成段落级写作闭环

## 5. 业务指标建议（对外沟通口径）

- **效率类**：平均写作准备时长、从检索到成稿段落时长
- **质量类**：可回溯引用率、用户有效反馈率（赞同/点踩）
- **使用类**：写作工具调用率、注入按钮使用率、图谱视图停留时长
- **工程/回归（对内也可对外简述）**：离线批量评测见 `scripts/eval/run_evaluation.py`，输出 `context_precision` / `faithfulness` 及按题类汇总均值（详见 [`docs/TECHNICAL_REPORT.md`](TECHNICAL_REPORT.md) §3.7.1 与 [`scripts/eval/README.md`](../scripts/eval/README.md)）

## 6. 阶段成果（简版）

- 实现“全局 Demo 复用”，降低多用户并发解析成本
- 建立“搜索-跳转-高亮”闭环，解决检索结果不可行动问题
- 完成“对话 -> 工具动作 -> 编辑器写入”链路，激活写作 Agent
- 构建写作舱多视图资源面板，显著提升沉浸式写作连续性
- 学术发现支持中文关键词自动英译检索；任务中心时间线可一键进入发现/写作/模拟评审与返修场景
- 顶栏 Agent 轻量 HUD + 流式对话占位优化，提升「等待可预期」与演示观感

## 7. 下一阶段规划（对外）

- 更精细化写作动作控制（选区/光标/文末）
- 引文质量评估与自动格式化（APA/GB/T）
- 多文献主题综述模板与自动大纲建议
